## Single Smart Agent Refactor

> Context: The dumb-vs-smart cost split is dead. Every handoff target is a capable fork of the same smart agent. "Complete and concise" is the key quality indicator for all handoffs — the receiver can act without re-reading history, and nothing unnecessary is included.

### 1. Delete `rqmd-dev.agent.md` — single agent only

`rqmd-dev.agent.md` is the vestigial "implementation sub-agent." Its execution contract is nearly identical to `rqmd.agent.md`; the only real differences are: it assumes someone else does planning, and it has the `agent` tool for sub-delegation. With a single smart agent model, all of that belongs in `rqmd.agent.md`. Delete the file, remove from `package.json` `chatAgents` if referenced, and verify no prompt references `agent: "rqmd-dev"`.

**Affects:** `agents/rqmd-dev.agent.md`, `package.json`

### 2. Purge dead `rqmd-ai` references from all skills

`rqmd-ai` was deleted from the CLI (`980fd5e`). At least 10 skills still reference commands that no longer exist:

| Skill | Dead reference |
|-------|---------------|
| `rqmd-implement` | `rqmd-ai --json --workflow-mode implement`, `rqmd-ai --json --dump-status proposed` |
| `rqmd-triage` | `rqmd-ai --json --dump-status proposed`, `rqmd-ai --json --dump-file` |
| `rqmd-brainstorm` | `rqmd-ai --json --workflow-mode brainstorm`, `rqmd-ai --json --dump-file`, `rqmd-ai --json --dump-status proposed` |
| `rqmd-export-context` | All examples: `rqmd-ai --json --dump-*`, `rqmd-ai --update`, `rqmd-ai --write` |
| `rqmd-status-maintenance` | `rqmd-ai --json --update`, `rqmd-ai --json --write --update` |
| `rqmd-docs` | `rqmd-ai --json --dump-id`, `rqmd-ai --json --dump-status` |
| `rqmd-doc-sync` | `rqmd-ai --json --dump-id`, `rqmd-ai --json --workflow-mode` |
| `rqmd-changelog` | `rqmd-ai --json --dump-status`, `rqmd-ai --json --workflow-mode` |
| `rqmd-bundle` | `rqmd-ai i --json`, `rqmd-ai install --json` |

| `rqmd-telemetry` | `rqmd-ai JSON output` in suggestion example |

**Replace with current CLI equivalents:**
- `rqmd --json --non-interactive` (base JSON output)
- `rqmd --status proposed --json --non-interactive` (filter by status)
- `rqmd --json --non-interactive <domain-token>` (filter by domain)
- `rqmd --update ID=STATUS --non-interactive` (status updates)
- `rqmd --verify-summaries --non-interactive` (unchanged)
- `rqmd init --scaffold` (init)
- `rqmd bug "title" --json` (bug filing)

**Remove entirely** (no CLI equivalent remains):
- `--dump-id`, `--dump-file`, `--dump-status`, `--dump-type` (removed flags)
- `--include-domain-markdown`, `--max-domain-markdown-chars` (removed)
- `--batch` mode (removed)
- `--write` (removed — `--update` now writes directly)
- `--workflow-mode` (removed)
- `rqmd-ai install/reinstall/upgrade` (removed — extension handles this now)

### 3. Remove "cheaper agent" and "dumb receiver" language everywhere

Prompts and skills still assume handoffs go to a less-capable session:

| File | Problematic language |
|------|---------------------|
| `prompts/refine.prompt.md` | "recommend cheaper implementation agent" |
| `docs/requirements/bundle.md` RQMD-EXT-046 | "paste a /go handoff prompt into a cheaper agent" |
| `docs/requirements/bundle.md` RQMD-EXT-047 | "defer to a cheaper implementation agent" |

Rewrite all handoff language to assume a fork of the same smart agent. The receiver is equally capable — handoffs exist for focus management, not cost optimization.

### 4. "Complete and concise" as explicit handoff standard

Define the quality indicator across all agent contracts, skills, and prompt handoffs:

- **Complete:** Receiver can make correct decisions without re-reading history or prior context
- **Concise:** Everything unnecessary is omitted; fits in a few focused lines

Apply to:
- `# Up next` closeout sections
- `/go` handoff prompts generated by `/brainstorm`, `/refine`, `/next`
- Skill handoff sections

**Target format:** 3 bullets max — ID + one-line state + open question if any.

### 5. Trim skills to focus contracts

Skills are written as instruction manuals for cold-starting a dumb agent. A smart fork already knows the domain. Each skill should become a single-screen focus declaration:

- **Charter:** What this skill narrows the agent to do (2-3 sentences)
- **Done-when:** Exit criteria (3-5 bullets)
- **Edge cases:** Non-obvious gotchas only

**Remove from skills:**
- Command rehearsals (`Start: rqmd-ai --json --workflow-mode implement`)
- Step-by-step workflows that duplicate the agent contract
- Metadata `examples` arrays referencing dead commands

**Keep:** Metadata `guide.summary` (one line) and any validation-specific commands that still exist.

### 6. Deprecate RQMD-EXT-025 and RQMD-EXT-026

These tracked distinct agent variants that no longer exist:
- **RQMD-EXT-025:** `rqmd-dev-longrunning` agent → behavior now expressed via `/go N` (slice count argument)
- **RQMD-EXT-026:** `rqmd-dev-easy` agent → behavior now expressed via `/go easy-win` (constraint argument)

Both agent files were already deleted. Mark both as 🗑️ Deprecated with note pointing to `/go` argument handling (RQMD-EXT-034).

### 7. Reconsider RQMD-EXT-047 — brainstorm resisting implementation

Current text: "defer to a cheaper implementation agent." This encodes the old split.

**Replace with:** The agent self-disciplines within one session — shapes ideas → promotes to tracked proposals → implements when ready. No hard session boundary or mandatory deferral to a separate session. The `resist implementation` instinct is still valuable (don't jump to code before the idea is shaped), but the mechanism is self-discipline, not a session handoff.

### 8. Rewrite RQMD-EXT-046 — auto-draft without "cheaper agent" framing

Current: "paste a /go handoff prompt into a cheaper agent."

**Replace with:** Requirements auto-drafted as 💡 Proposed so any session (current or forked) can immediately work them. The `/go` handoff prompt is still generated for focus management, but the framing is "here's a focused work package" not "paste this into a cheaper chat window."

### 9. Rethink `rqmd-export-context` skill

The skill's entire API surface was removed from the CLI: `--dump-id`, `--dump-status`, `--dump-file`, `--include-domain-markdown`, `--update`, `--write`. The remaining CLI surface is:
- `rqmd --json --non-interactive` (full catalog)
- `rqmd --status <status> --json --non-interactive` (status filter)
- `rqmd <domain-token> --json --non-interactive` (domain filter)

The skill needs fundamental rewrite or deprecation. With a smart agent that can read files directly, the primary "export" mechanism is just reading `docs/requirements/*.md`. Consider:
- Simplify to "read requirement docs directly; use `rqmd --json` for machine-readable overview"
- Or deprecate entirely if the agent's native file reading is sufficient

### 10. Delete `rqmd-bundle` skill entirely

`rqmd-bundle` is a legacy skill from the `rqmd-ai` days. Every command it references is gone (`rqmd-ai install`, `--bundle-preset`, `--dry-run`, `--overwrite-existing`, `--answer`, `--chat`). Bundle installation is now handled by the VS Code extension itself — there is no user-facing "install bundle" workflow to teach.

**Action:** Delete `skills/rqmd-bundle/` directory, remove from `package.json` `chatSkills`, and remove from any agent skill lists.

### 11. Merge init skills into one `rqmd-init` with interview preserved

Both `rqmd-init` and `rqmd-init-legacy` reference `rqmd-ai init --chat --json` — gone. But the **interview workflow itself is magical** for onboarding existing barely-documented projects. The structured multi-choice interview with smart defaults, `/dev` and `/test` skill generation, and requirement catalog seeding should absolutely stay as a core feature.

**Action:**
- Delete `skills/rqmd-init-legacy/` entirely; merge any unique "existing repo" guidance into `rqmd-init`
- Remove from `package.json` `chatSkills`
- Rewrite `rqmd-init` per focus-contract format:
  - **Charter:** Onboard a new or existing project into rqmd using the CLI scaffold + agent-driven interview. The agent runs `rqmd init --scaffold` (or command palette), then drives the structured interview: project context discovery, multi-choice questions with smart defaults, `/dev` and `/test` skill generation, starter requirement catalog seeding.
  - **Done when:** Starter requirement docs exist and summaries verify; `/dev` and `/test` skills reviewed and customized for the repo's actual commands; generated requirements treated as starting point.
  - **Edge cases:** For repos with existing docs/backlog, inspect repo structure and `gh issue list` when available before seeding; don't overwrite existing requirement docs.
- Keep the `metadata.legacy_init` block (max_domain_files, max_issue_requirements, etc.) — it's useful agent-facing config for the interview
- Purge dead `rqmd-ai init` command references; replace with current CLI entrypoints

### 12. Clean up `rqmd-status-maintenance` for current CLI

References `rqmd-ai --json --update` and `rqmd-ai --json --write --update`. The current CLI uses:
- `rqmd --update ID=STATUS --non-interactive`
- `rqmd --update-priority ID=PRIORITY --non-interactive`
- `rqmd --update-flagged ID=true --non-interactive`
- Add `--dry-run` for preview

Straightforward command updates.
